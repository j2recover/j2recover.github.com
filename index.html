AIX JFS2 Recovery

We are providing jfs2 recovery software and jfs2 recovery services for AIX. j2recover.com has been involved in all types of incidents related to deleted, missing or inaccessible jfs2 files.

Both j2undelete and j2recover can be used to recover jfs2 file.

Here is the recommendation for how to select them:

Recommendation Scenery Reason
j2undelete

The deleted jfs2 file is still used by some process.

 Since the deleted jfs2 file is still used, it will not be overwritten, thus umount is needless.
In such case j2undelete is the best selection.
j2recover

The deleted jfs2 file is not used, and umount is allowable

 In such case we should firstly umount fs to avoid data overwriting.
Thus j2recover is the best selection.
j2undelete The deleted jfs2 file is not used, but umount is not allowable, and "j2recoverlist –f" can detect the deleted file In such case we have the risk that the deleted file may be overwritten.
Since j2undelete has most short recovery time, it is the best selection to reduce the data overwriting chance.
If j2undelete has alreay been installed in the system and it is used immediately after deletion, we can effectively minimize the the data overwriting chance.
j2recover The deleted jfs2 file is not used, but umount is not allowable, and j2recoverlist can detect the deleted file The full disk scanning of j2recover can recover all deleted files as long as they are not overwritten in the disk.
Even umount is not permitted, j2recover can recover deleted files and copy them to specified place.
